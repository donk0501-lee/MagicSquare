## 1. 작업 요약 (결과)

- **경계 계약(I1~I4) 검증 + 고정 오류 스키마:** `MagicSquareContractError(code,message)`로 PRD §8.1 문자열 그대로 노출.
- **도메인 규칙(FR-02~FR-05) 구현:** 빈칸 좌표(row-major, 1-index), 누락 두 수, 마방진 판정(10선 합 34), 두 배치 시도(성공 시 `int[6]`).
- **GUI(MVP) 추가:** `python -m magic_square`로 PyQt6 창 실행(4×4 입력, “풀기” 버튼, 결과 표시/오류 메시지).
- **Formal Test Cases 회귀:** `docs/Formal_Test_Cases.md`의 주요 케이스를 pytest로 추가(`tests/test_formal_contract.py`).

## 2. 커밋 단위 작업 내역

### 2.1 커밋 `a2dccce` — 첫 Dual-Track GREEN (2개 TC)

- **선택 RED**
  - Track B: `L-RED-01` — `find_blank_coords()` row-major 2좌표(1-index) 반환
  - Track A: `U-RED-01` — 비 4×4 입력 시 `SIZE_INVALID` 고정 message
- **GREEN**
  - `src/magic_square/logic.py`: `find_blank_coords` 구현
  - `src/magic_square/boundary.py`: 4×4 크기 검증(행·열)
  - `src/magic_square/constants.py`: `MATRIX_SIZE` 등 상수 단일 출처 도입

### 2.2 커밋 `e79a7ef` — GUI 실행 가능 + 전체 solve 경로 연결

- **도메인/경계 구현 확장**
  - Domain: `find_not_exist_nums`, `is_magic_square`, `solution` 구현
  - Boundary: I1~I4 검증 및 `NOT_MAGIC`(도메인) → `NOT_MAGIC_SQUARE`(경계) 매핑
- **PyQt6 Screen(MVP)**
  - Qt import는 `src/magic_square/gui/` 하위에만 위치
  - 공식 진입점: `python -m magic_square` (`src/magic_square/__main__.py`)
- **의존성/문서**
  - `pyproject.toml`: optional extra `gui = ["PyQt6>=6.5"]`
  - `README.md`: GUI 실행 1줄 문서화

### 2.3 커밋 `7a0726b` — PRD §11 탐지 순서 준수 + Formal TC 추가

- **PRD §11 탐지 순서 고정:** I1(SIZE) → I2(BLANK_COUNT) → I3(VALUE_RANGE) → I4(DUPLICATE)
  - “다중 위반” 입력에서 **빈칸 위반이 값 범위보다 먼저** 탐지되도록 수정
- **예상 밖 입력 처리(명세 선택)**
  - `None`/`[]` 등 “4×4로 볼 수 없는 구조”는 `SIZE_INVALID`로 처리
  - 비정수 셀(예: 문자열)은 경계에서 `INTERNAL_ERROR` / `Unexpected error.`로 래핑(문서 `FTC-MS-D-005`의 ‘구현 선택’ 항목을 테스트로 고정)
- **도메인 판정 제약 강화**
  - `is_magic_square`는 **0이 존재하면 False** (Formal 문서 §0 “완성 격자에만 적용”)
- **Formal TC 회귀 파일 추가**
  - `tests/test_formal_contract.py`: FTC-MS A/B/C/D/E 및 U-09, NFR-04, 탐지 순서 회귀 포함
